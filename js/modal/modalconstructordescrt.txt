Initialization:
let modal = new ModalConstructor(triggerSelectorOrEl, options)

By default, window opening should be configured forcefully with the modal.open() method. If you need the window to open automatically when clicking on the trigger (for example, if the content is known and will not change, or if there are no other actions besides opening on the trigger), you need to add the autoOpen: true option.

--triggerSelectorOrEl - string, selector or element on click of which the window opens. (If the window is static (hidden in markup), for the trigger, add - data-path = "value", for the content - data-modal="value");
--options = object with settings.

Options:
-- autoOpen: (boolean, default=false) specifies whether the window should be opened automatically by the plugin. Default is false, modal.open() method should be used to open the window.
-- isStatic: (boolean, default=false) specifies whether the modal window is static or dynamic;
-- modalInner: (html/el, default = null), content of the modal window when dynamically creating it. (The constructor creates an overlay, content wrapper, close button, and the content itself can be added in this setting);
-- enableTransition: (boolean, default = true) if true, transition for window appearance is set. If false, animation can be set using a library (for example, GreenSock). In the case of external animation for a dynamic window, it is mandatory to specify animTime, which will be a delay when removing from the DOM (the value should be coordinated with the animation time of the library)
-- animTime: (ms, default=null) animation time. This parameter should be specified when setting up animation using a library, as it also serves as a delay for removing the modal from the DOM
-- easing: (string, default = 'ease') name of the easing function for transition
-- elemToFocus: (string, default=null) pass the selector of the element that should be focused when opening the modal window. If not set, focus is set to the first focusable element in the modal

//classes
-- modalCloseBtnClass: (string, default-null) class of the close button (if any) (for dynamic windows, it also needs to be created independently)
-- modalOverlayClass: (string, default="modal-overlay"), class of the overlay, default class is added when dynamically created, for static windows, appropriate classes need to be passed
-- modalWrapperClass: (string, default="modal-wrapper"), class of the content wrapper
-- modalAnimationClass: (string, default ="modalInitAnim"), class of the initial animation state.
-- modalInAnimClass: (string, default="openAnim"), class of the window opening animation,
-- modalOutAnimClass: (string, default="outAnim"), class of the window closing animation,

Methods:
-- modal.updateInner(html-string/el) - can replace/add content of a dynamic modal, at the moment of clicking the trigger
-- modal.open() - opens the modal
-- modal.close() - closes the modal

Events:
Allow doing something with the window at the moment of opening/closing (for example, animation). Events bubble up, can be caught on document, e.target = modal window
-- modalOnOpen
-- modalOnClose

Example html for a static window:
<button class="trigger" type="button" data-path="1">open </button>

  <div class="overlay" data-modal="1">
    <div class="content-wrapper" >
      <button class="close-btn" type="button">close</button>
  </div>
  </div>
Notes:
! If the window is dynamic, when creating its content, consider the close button if needed and pass its class in modalCloseBtnClass. By default, the window is without a button.

!! If there are elements with position: fixed in the document, they need to be assigned an additional class "fixed-el". This is necessary to remove the "jump" of content when opening the modal and blocking scroll.

!!! If normalize.css is not used in the project, it is necessary to reset the default margin of the body, if this is not done, the width of the scrollbar is incorrectly considered and the "jump" of the content is preserved.

!!!! When opening the window, focus will be on the first focusable element in the markup (for dynamic windows, this is the content wrapper). To set focus to the desired element, pass the corresponding selector to elemToFocus.
